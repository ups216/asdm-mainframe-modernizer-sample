# ASDM Action: Git Commit

## Overview

This action commits changes to the current branch with a commit message generated in the current session.

## Purpose

- Automate the git commit process
- Use standardized commit messages generated by the AI engine
- Maintain consistency in version control

## Steps

1. Retrieve the generated commit message from the current session
2. Stage all changed files (or specific files based on user selection)
3. Create a git commit with the generated message
4. Provide confirmation of the commit

## Usage

This action is typically invoked after git changes have been made and a commit message has been generated.

## Prerequisites

- Working git repository
- Staged or unstaged changes
- Generated commit message from `asdm-git-commit-message` action

## Input

- Commit message (generated from current session)
- Files to commit (all staged files or user-selected files)

## Output

- Commit SHA
- Commit confirmation message
- Branch information

## Configuration

Refer to [git-commit-spec.md](../spec/git-commit-spec.md) for commit specifications and standards.
