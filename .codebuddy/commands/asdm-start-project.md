# ASDM Action: Start Project

## Overview

This action detects the current workspace context and starts the project in development mode using context generated by Context Builder toolset.

## Purpose

- Detect the current Agentic Engine provider
- Analyze the current workspace structure using context from `.asdm/contexts/`
- Initialize development environment based on context information
- Start the project with appropriate development tools specified in context
- Provide feedback on the startup status

## Steps

1. **Load Workspace Context**: Read and analyze context from `.asdm/contexts/index.md` and other context files
2. **Analyze Project Type**: Identify the project type (Node.js, Java, Python, etc.) from context files
3. **Initialize Environment**: Set up development environment based on context specifications
4. **Start Development Server**: Execute the appropriate startup command from context
5. **Provide Feedback**: Return status message and development server URL from context

## Usage

This action is typically invoked from the Agentic Engine provider's entry point as a command shortcut. It relies on context previously generated by the Context Builder toolset.

## Input

- Current workspace directory
- Project context from `.asdm/contexts/` directory
- Context Builder toolset must be installed and context must be built

## Output

- Project started in development mode
- Development server URL (from context)
- Status message confirming successful startup
- Context validation results

## Context Integration

This action uses the following context files from Context Builder:
- `.asdm/contexts/index.md`: Workspace overview and structure
- `.asdm/contexts/standard-project-structure.md`: Project organization and technology stack
- `.asdm/contexts/deployment.md`: Development and deployment configuration
- Other context files as needed for specific project types

## Prerequisites

- Context Builder toolset must be installed
- Context must be built using `/asdm-context-build` command
- Context files must exist in `.asdm/contexts/` directory

## Workflow

1. **Check Context Availability**: Verify context files exist in `.asdm/contexts/`
2. **Parse Context**: Extract project type, startup commands, and dependencies
3. **Execute Startup**: Run the appropriate development server based on context
4. **Validate**: Confirm project started successfully and report status

## Error Handling

- If context files are missing, prompt user to run context build first
- If project type cannot be determined from context, ask for clarification
- If startup command fails, provide troubleshooting information from context files
